# 项目说明书

## 1. 项目概述

本项目是一个轻量级库存管理系统，目标是提供一个可运行的库存管理 MVP（最小可用产品），支持商品的增删改查、库存状态判断、低库存提示，以及一个可直接操作的前端页面。

当前仓库由两部分组成：

- 后端：`Flask + SQLite`
- 前端：`Vite + 原生 JavaScript`

项目适合用于课程作业、小型内部工具、原型验证或后续二次开发的基础版本。

## 2. 这个项目要完成哪些东西

结合当前代码和文档，这个项目的目标任务可以分为以下几类：

### 2.1 基础业务功能

- 商品信息管理：新增、查询、修改、删除商品
- 商品字段管理：`SKU`、商品名称、库存数量
- 库存状态判断：
  - 库存为 `0` 时标记为 `out`
  - 库存为 `1-4` 时标记为 `low`
  - 库存大于等于 `5` 时标记为 `ok`
- 低库存提示：当库存低于 `5` 时给出补货提醒

### 2.2 接口能力

- 提供健康检查接口
- 提供商品列表接口
- 支持按关键字搜索商品
- 支持分页查询
- 提供单个商品详情接口
- 提供新增、更新、删除接口
- 对写操作增加 API Key 保护

### 2.3 页面能力

- 展示库存概览统计
- 支持搜索商品
- 支持分页查看商品
- 支持录入商品
- 支持编辑商品
- 支持删除商品
- 支持按 ID 获取单个商品详情
- 支持在浏览器中保存 API Key

### 2.4 工程化基础

- 支持环境变量配置
- 支持数据库初始化
- 支持基础日志
- 支持 API 测试
- 支持本地前后端联调

## 4. 还没有完成或仍待完善的内容

如果把这个项目定义为“库存管理 MVP”，那么核心目标基本已经完成。

但如果把它定义为“可投入真实业务使用的完整库存系统”，目前还有以下内容没有完成，或者仓库中还没有体现：

### 4.1 业务能力层面

- 没有入库、出库、调拨、盘点等库存流水功能
- 没有库存变更历史和审计记录
- 没有商品分类、品牌、供应商、仓库等扩展字段
- 没有采购、补货单、订单联动等业务流程
- 没有预警通知能力，目前只有页面上的低库存提示

### 4.2 权限与安全层面

- 没有用户登录系统
- 没有角色与权限管理
- 只有单一 API Key，适合开发或内部简单使用，不适合多人生产环境

### 4.3 工程化与交付层面

- 没有 CI/CD 配置
- 没有 Docker / 容器化部署方案
- 没有数据库迁移体系，当前只有较轻量的初始化与兼容处理
- 没有前端自动化测试
- 当前仓库中没有安装前端依赖，因此前端构建结果未在本次检查中实际执行验证

### 4.4 产品体验层面

- 页面为基础管理界面，适合原型或课程项目，离成熟后台系统还有差距
- 没有批量操作能力
- 没有导入导出能力
- 没有图表分析或报表下载
- 没有移动端专门优化设计
